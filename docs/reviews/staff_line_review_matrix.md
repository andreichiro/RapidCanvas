## Cross-Cutting Test Strategy

Every implementation slice must introduce or update a test family that catches the new behavior as a generalized property:

| risk area | generalized test strategy |
| --- | --- |
| Source quality and citation eligibility | Unit tests score documents across thread, quote, linked primary, fetched web, snippet-only, stale, off-topic, catalog, and injection cases; integration tests verify only eligible sources become bullet citations. |
| Claim support and usefulness | Deterministic support tests map bullets to source text by material entities/key terms and reject unsupported named entities, dates, causal claims, broad history, and snippet-only sole support. |
| Live-quality scoring | Eval tests fail rows with off-topic cited sources even when bullet/citation counts look healthy; report tests require every failed row to carry a concrete reason. |
| Image evidence | Unit tests cover alt-text fallback, vision metadata, malicious alt/vision text, empty image evidence, video warnings, and image-required expected point recall. |
| Qdrant/request isolation | Concurrent retrieval tests run two namespaces with overlapping documents and assert no deletion, no cross-contamination, deterministic ordering, explicit fallback trace, post-scoped namespace handoff, and static guards against shared collection recreation/deletion. |
| Bluesky parsing | Parser/normalizer tests cover URL variants, DID/handle validation, facets, links, quotes, images, videos, malformed timestamps, indexed-time fallback, unavailable records, and warnings. |
| Robots/safe fetch | Fetch tests block private/local/file/link-local URLs, respect robots disallow as best effort, redact warnings, mark snippet-only evidence, and survive search failures. |
| Frontend runtime safety | Component tests mock cancel, malformed JSON, structured API errors, provider skipped/runnable states, source-quality badges, vector backend, image/video warnings, and long text wrapping. |
| Ops and docs truth | Command tests or command-review checks verify deep review, cached eval, Docker preflight, workflow syntax, secret hygiene, and requirement statuses tied to code/tests/eval/docs. |
| Staff review closure | Ledger tests discover the scoped production surface, require explicit P0 files, reject duplicate rows, reject empty cells, and fail `make deep-review` when any production file lacks a review row. |
